# Incubation Checklist

- [x] Have achieved and maintained an OpenSSF Best Practices Badge at the ‘Passing’ level.
- [x] Have an open and documented technical governance, including:
   - [x] A README file in each code repository, welcoming new community members to the project and explaining why the project is useful and how to get started (follow the guidelines at the README checklist to create an excellent README file).
   - [x] A GOVERNANCE file that documents the project’s technical governance (template).
   - [x] All current Technical Steering Committee members specified in the ‘Technical Steering Committee (TSC)’ committee in LFX Project Control Center (details on how this should be set up are here).
   - [x] A CODEOWNERS or COMMITTERS file to define individuals or teams that are responsible for code in a repository, as well as documenting current project owners and current and emeritus committers ( template )
- [x] Documentation of the architecture (aka high-level design) of the software produced by the project (aligns with this OpenSSF Best Practices Silver badge requirement).
  - [ ] The project should work with the Archimate SIG to ensure the project’s functional architecture is built in the LF Energy ArchiMate tool.
- [ ] A documented roadmap that describes what the project intends to do and not do for at least the next year (aligns with this OpenSSF Best Practices Silver badge requirement). See [Roadmap](https://github.com/arras-energy/tsc/blob/main/planning/Roadmap.md).
- [x] Community and contributor growth assessment using the data from LFX Insights. Specific metrics that will be reviewed are.
  - [x] Current number of contributors, committers, and different organizations contributing to the project.
  - [x] Flow of commits / merged contributions
- [ ] A credible plan for developing a thriving user community, particularly expanding the number of committers and contributors to the project. See [Community Building Plan](https://github.com/arras-energy/tsc/blob/main/planning/Community_Building.md).
- [x] An outline of the plan for the project to complete the requirements for the Early Adoption stage. See [Early Adoption Plan](https://github.com/arras-energy/tsc/blob/main/planning/Early_Adoption.md).
